Benny's Recipes

Welcome to Benny's Recipes

This is a personal digital recipe book. Recipes are stored in a GitHub Gist — a free file hosting service — so they're backed up and accessible on any device.

What you'll need

A free GitHub account and a personal access token with Gist permissions.

First-time setup

  1. Create a free account at github.com if you don't have one already.
  2. Go to gist.github.com and click + to create a new Gist. Set the filename to recipes.json and the content to []. Set it to Secret, then click Create secret gist.
  3. Copy the long ID from the URL — it looks like: a1b2c3d4e5f67890abcd. That's your Gist ID.
  4. Go to github.com/settings/tokens, click Generate new token → Classic. Give it a name, set no expiry, and tick only the gist checkbox. Click Generate token and copy it — you'll only see it once.
  5. In this app, tap the 🔑 key icon in the header, paste your Gist ID and token, then tap Save & connect.
Your credentials are saved only on this device in the browser's local storage. They are never sent anywhere other than GitHub's servers.

Setting up on a new device

Each device needs to be connected separately. Tap the 🔑 key icon and enter the same Gist ID and token. Your recipes will load straight away.

Keep your Gist ID and token in a password manager so you can reconnect on new devices easily.

Adding a recipe

Tap + Add recipe in the header. Fill in the name, tags, cook time, and serves. You can add a source URL or a YouTube link — a YouTube link will appear as a playable video inside the recipe.

Ingredients and method sections

Ingredients and method steps are added directly — no section wrapper by default. Use the + Ingredient and + Step buttons to build your list. If you want to group items, tap + Section to create a named section. Sections must have a name. Unnamed items always appear at the top.

Collapsing sections while editing

Each ingredient or method section has a ⌄ chevron in its header. Tap it to collapse the section into a compact dot-point summary — each ingredient shows its name, amount, unit, prep, and any active toggles. Tap again to expand. Useful for keeping long recipes manageable while editing.

Drafts and publishing

Every new recipe starts as a Draft, hidden from the main page until you're ready. When viewing a recipe, tap the status icon next to the recipe name — a green tick means published, an amber dashed circle means draft. You can also publish or unpublish from inside the Edit form using the Publish/Unpublish button in the footer. Toggle the Drafts switch in the toolbar to see your unpublished recipes.

Organising with tags

Tags categorise your recipes — things like "Dinner", "Italian", or "Bake". Type any tag when editing a recipe. Custom tags you create appear under Other in Settings → Tags, where you can move them into the appropriate category.

Serves

Set an exact number of serves, or switch to Range mode to enter a minimum and maximum — useful for recipes that comfortably feed different numbers depending on appetite.

Ticking off while cooking

While reading a recipe, tap any method step or ingredient to mark it as done — it fades and gets a strikethrough. Tap again to un-tick. The state resets when you close the recipe.

Editing and deleting a recipe

When viewing a recipe, tap the ✎ pencil icon in the header to open the edit form. From there, the 🗑 bin icon in the footer moves the recipe to the bin — accessible via the bin icon in the main header where you can restore or permanently delete it.

Undoing changes

While editing a recipe, the ↩ ↪ undo/redo buttons in the edit form header track your changes within that editing session — adding or removing ingredients, editing fields, toggling scaling flags, and more. Changes are not undoable once you've saved and closed the form.

Scaling a recipe up or down

When viewing a recipe, use the 1× / ½× / 2× / 3× scale buttons — they scroll with the recipe content — to change the serving size. Ingredient quantities update automatically, with units adjusting sensibly — for example, 3 teaspoons becomes 1 tablespoon.

Ingredients that don't scale well

Some ingredients — like salt, spices, or baking powder — don't simply double or halve. Tap the ⊘ button next to any ingredient when editing to set a specific amount for each scale size individually. Leaving a scale size blank means proportional scaling is used for that size.

Method steps that change with scale

The same ⊘ flag works for method steps — useful when cooking times or temperatures need different instructions at different batch sizes.

Turning off scaling for a recipe

When editing, the Recipe can be scaled toggle lets you disable scaling entirely for recipes where it doesn't apply. The scale controls won't appear when viewing that recipe.

Adding substitutes

Any ingredient can have one or more substitutes. When editing an ingredient, tap the ⇄ button to open the substitutes panel. Each substitute has its own name, amount, unit, prep note, and an optional Reason field — use this to explain why the substitution might be made (e.g. "dairy-free alternative" or "if unavailable").

Inherited fields

When you add a substitute, its amount, unit, prep, and note fields default to the parent ingredient's values. The parent's values appear as placeholder text in each field. If you leave a field blank, the substitute will show the parent ingredient's value in the recipe view — useful when only the name changes (e.g. swapping "brown onion" for "red onion" with the same prep and quantities). Fill in a field to override just that value.

Choosing a substitute when viewing a recipe

Ingredients that have substitutes show a ⇄ button in the recipe view. Tapping it opens a small chooser listing the original ingredient and all available substitutes. If a substitute has a reason, it appears in smaller text beneath the name to help you decide. Selecting a substitute swaps that ingredient's name, quantity, and prep note in the table. Tap the button again and choose Original recipe to revert.

Scaling substitutes

Substitutes support the same scale overrides as regular ingredients. Use the ⊘ button within the substitute panel to set specific amounts for ½×, 2×, or 3× scale if the substitute doesn't scale proportionally.

Multiple substitutes

An ingredient can have more than one substitute — for example, a recipe calling for cream might offer both a coconut cream substitute and an oat cream substitute. Only one substitute can be active at a time per ingredient.

Substitute groups — recipe modes

When editing a substitute, you can assign it a Group using the dropdown (e.g. "Dairy Free", "Vegan"). When any substitutes have a group set, a Mode bar appears above the ingredients when viewing the recipe. Tapping a mode (e.g. "Dairy Free") activates all substitutes tagged to that group at once — swapping every matching ingredient simultaneously. Tap Original to revert.

Searching for a recipe

Use the search bar at the top to find recipes by name or tag. On a phone, tap the magnifying glass icon to bring up the search field.

Filtering your recipes

Tap Filter in the toolbar to narrow down which recipes are shown. You can select multiple options. Within the same category (e.g. "Breakfast" and "Dinner") the app shows recipes matching either. Across different categories (e.g. "Korean" and "Dinner") only recipes matching both are shown. Active filters appear as removable tags below the toolbar.

Filtering by cook time or serves

The filter panel includes cook time and serves ranges. Select one or more ranges to narrow results. Only ranges that match at least one of your recipes are shown.

Grouping recipes

Tap Group in the toolbar to organise your recipes into labelled sections on the main page. A recipe tagged with both "Dinner" and "Lunch" will appear under both headings.

Downloading your recipes

Tap the ↓ icon in the header to open the export panel. A checklist of all your recipes appears — tick the ones you want to include, use the All or None buttons to select quickly, then choose one or more formats to download.

Available formats

Text file (.txt) — a plain, human-readable file listing every recipe with ingredients, substitutes, and method steps. Easy to open anywhere, including your phone's Notes app.

Markdown (.md) — the same content formatted with headings and bold text. Opens nicely in any markdown viewer, Notion, GitHub, or Obsidian.

Spreadsheet (.csv) — one row per recipe with columns for name, tags, cook time, serves, source, notes, ingredients, and method. Opens in Excel or Google Sheets.

Interactive HTML (.html) — a single self-contained file for one recipe. Includes scale buttons, substitute toggles, and note reveals. Ideal for sharing a recipe with someone who doesn't use the app — just open in any browser.

Full Gist backup (.json) — a complete snapshot of your Gist files (recipes, bin, and settings). Use this to migrate to a new Gist or restore everything from scratch using the same version of the app.

Automatic backups in your Gist

Every time you save a recipe, the app also writes backup.txt and backup.csv files directly to your Gist. These update automatically so you always have a recent, human-readable copy available — no manual download needed.

Restoring from a Gist backup

If you ever need to start fresh, create a new Gist and create three files: recipes.json, bin.json, and settings.json. Paste in the contents from the corresponding entries inside your downloaded gist-backup.json file. Then connect the app to the new Gist using the 🔑 key icon.

Adding to your iPhone home screen

  1. Open the app in Safari (must be Safari, not Chrome or another browser).
  2. Tap the Share button — the square with an arrow pointing up — at the bottom of the screen.
  3. Scroll down and tap Add to Home Screen.
  4. Give it a name and tap Add. The app will appear on your home screen and open full-screen.

Connecting on a new iPhone

  1. Open the app and tap the 🔑 key icon in the header.
  2. Enter your Gist ID and personal access token — the same values used during setup.
  3. Tap Save & connect. Your recipes will load straight away.
Every device needs to be connected separately. Credentials are not shared between devices automatically.

Using without internet

If you lose your connection, the app continues working from the last loaded version of your recipes. You can still browse, search, and edit — changes are saved locally and synced to GitHub automatically when you're back online. A banner at the top indicates when you're offline.

If recipes aren't loading on a new device, the most likely cause is missing credentials. Tap the 🔑 key icon and enter your Gist ID and token.
